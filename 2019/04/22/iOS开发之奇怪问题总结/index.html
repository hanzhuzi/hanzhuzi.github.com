iOS开发之奇怪问题汇总

2019-04-22
一、项目中使用Pod安装或者更新时报无法找到Pods_Project_name.framework或者.a或者无法找到某些头文件错误。

1、打开xx.xcworkspace，当项目中的Pods.xcodeproj的Build Active Archive Only设置为YES，而项目工程里设置的是NO，也就是不一致，那么就有可能会导致报 ‘ld: framework not found xxx.framework或者.a’错误或者报一些头文件找不到的类似错误。
解决方法：将Pods.xcodeproj的Build Active Archive Only的设置和工程里的设置保持一致重新编译，若还是会报错，那么就找到DriveData目录然后将里面的文件全部删掉，注意！只删DriveData目录下对应工程路径下的文件即可，然后再重新Build一般就能编译成功了。

四、UITableView，UICollectionView的cellForRowAtIndexPath方法不调用问题。

出现这种问题一般是UITableView，UICollectionView的宽度或者高度为0导致的，或者代理方法没有写对，代理要设置，cell的高度要返回，cell的个数要返回，这样就可以保证代理方法调用了。

五、推送相关问题

1、极光推送上传p12文件时，导出p12文件不能展开证书进行导出，因为展开导出的是私钥，直接导出”xxxx_apns.cer”的p12文件。
2、极光推送的demo有App扩展，扩展的bundleID应该设置为主工程的bundleID后面加个后缀xxx.sub即可。

七、git相关

修改密码可能需要重新验证，或者将密码改回原来的密码即可。再则就重新clone项目。

八、支付宝接入问题

1、自己签名时需要导入openssl库，导入时一般会报asn1.h not found错误
解决：将openssl导入后，设置Headers Search Path 添加openssl的上一层目录即可解决。
2、支付宝的demo会误导我们加签，当出现系统繁忙 AL000xx之类的错误时就是加签出现了问题。
注意：加签时是对非encode的订单信息进行加签，而不是encode的订单信息加签，这个有误导！

智能合并
1.使用git checkout 将根据A分支创建一个A_temp分支，避免影响A分支

2.将B分支合并到A_temp分支

3.切换到A分支，并使用git checkout 将A_temp分支上的系统消息功能相关文件或文件夹覆盖到A分支

另外如果只想将feature分支的某个文件f.txt合并到master分支上。
1: git checkout feature
2: git checkout –patch master f.txt
第一个命令： 切换到feature分支；
第二个命令：合并master分支上f文件到feature分支上，将master分支上 f 文件追加补丁到feature分支上 f文件。你可以接受或者拒绝补丁内容。

如果只是简单的将feature分支的文件f.txt copy到master分支上；
